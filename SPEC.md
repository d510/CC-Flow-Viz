# cc_flow_viz 仕様書

作成日: 2026-03-21
最終更新: 2026-03-21（実装反映済み）

---

## 概要

コールセンターの1日（9:00〜18:00受付・対応完了まで稼働）の運営を
早回し再生するレトロゲーム風ビジュアライゼーション。

- 入電 → 待機 → 対応 → 完了／放棄 → 再入電 の流れをドットで表現
- ポートフォリオ用途：エンジニア・採用担当・コールセンター業務経験者の両方に訴求
- VPSサーバー上に静的ファイルとして配信

## シミュレーション設定

| 項目 | 値 |
|---|---|
| 受付時間 | 9:00 〜 18:00（新規入電受付） |
| 稼働終了 | 18:00以降は新規入電停止。待機中の顧客がいなくなるまで対応継続 |
| 再生時間（実時間） | 約200秒（3分20秒）※1x速度・受付終了まで |
| 時間スケール | 1秒 ≒ 2.7分（シミュレーション時間）= 9時間÷200秒 |
| オペレータ総数 | 30名（新人10名 / ベテラン20名） |
| キュー最大長 | 30名（満杯時は即放棄） |
| 最大待機時間 | 5分超（シミュレーション時間）で放棄 |
| 再入電確率 | 放棄者の30% |
| 再入電までの待ち | 5〜15分後（シミュレーション時間） |
| 乱数シード | 固定（seed=42、毎回同じ展開） |
| デフォルト入電数 | 850件/日（`DEFAULT_CALLS` 定数で管理、UIから変更可） |

## 再生速度

| ボタン | 倍率 | 1x比 |
|---|---|---|
| 1x | 1倍（デフォルト） | 約200秒で9時間分 |
| 5x | 5倍 | 約40秒で9時間分 |
| 10x | 10倍 | 約20秒で9時間分 |
| 100x | 100倍 | 約2秒で9時間分 |

### 速度独立性

サブステップ方式（固定刻み TICK=0.25シミュ分）でシミュレーションを進める。
どの速度で再生しても放棄判定・対応完了・スポーンのタイミングが一定になり、
**再生速度を変えてもシミュレーション結果（応答率・件数など）は変化しない。**

```javascript
update(dtReal, speedMul) {
  const TICK    = 0.25;  // 固定刻み
  let remaining = dtReal * BASE_SIM_MIN_PER_SEC * speedMul;
  while (remaining > 0 && !finished) {
    _tick(Math.min(TICK, remaining));
    remaining -= TICK;
  }
}
```

## スタッフ構成

| タイプ | 人数 | 時給 | CPH（件/時） | 対応時間の目安 | 休憩時間 |
|---|---|---|---|---|---|
| 新人（N） | 10名 | 1,500円 | 2〜4 | 15〜30分/件 | 30分 |
| ベテラン（V） | 20名 | 2,000円 | 5〜7 | 8〜12分/件 | 1時間 |

※ 勤務時間は全員共通（9:00〜18:00）。新人・ベテランの違いは時給・CPH・休憩時間のみ。

各オペレータのCPHは初期化時にランダム決定・固定（乱数シードで再現可能）。

### 1日の人件費上限（休憩時間を除いた稼働時間ベース）

- 新人: 10名 × 1,500円 × 8.5時間（9時間 - 30分休憩）= 127,500円
- ベテラン: 20名 × 2,000円 × 8時間（9時間 - 1時間休憩）= 320,000円
- **合計: 447,500円/日**

※ 人件費は休憩中を除く実稼働時間（`workMin`）× 時給で積算するため、上記は理論上限値。

コスト計算は休憩中を除いた実稼働時間（`workMin`）× 時給で積算。

### 休憩ルール

- 休憩帯: 12:00以降に順番にローテーション（上限時刻なし）
- 同時休憩: 最大6名まで
- 新人（パートタイム）: 30分休憩
- ベテラン（フルタイム）: 1時間休憩
- 12:00時点で受電中だったオペレータは通話終了後に休憩に入る（休憩チェックが割り当て処理より先に実行されるため）

### オペレータ割り当て方式

空きオペレータの中から**最長待機優先**（最も早く空いたオペレータを選ぶ）で割り当てる。
`idleSince`（idle になった時刻）が最小の operator を `reduce()` で選択。

## 時間帯別入電パターン

1日の総入電数を100%としたときの時間帯別割合。

| 時間帯 | 割合 | 備考 |
|---|---|---|
| 09:00〜10:00 | 25.0% | 朝ピーク |
| 10:00〜11:00 | 15.0% | 午前安定 |
| 11:00〜12:00 | 12.0% | 午前後半 |
| 12:00〜13:00 | 14.0% | 昼（少し回復） |
| 13:00〜14:00 | 11.0% | 午後前半 |
| 14:00〜15:00 | 8.0% | 午後中盤 |
| 15:00〜16:00 | 6.0% | 午後後半 |
| 16:00〜17:00 | 5.0% | 夕方 |
| 17:00〜18:00 | 4.0% | 終業前 |

スポーンはスケジュール方式。初期化時に全スポーン時刻を生成・ソートし、
`_tick()` ごとに `simMin` を超えたエントリを処理する。

## 顧客とドットの状態

### 顧客の概念

| 概念 | 説明 |
|---|---|
| ユニーク顧客数 | 実際の人数（1人が複数回かけても1カウント） |
| 総コール数 | 再入電を含む実際の入電回数 |

### ドットの状態と色

| 色 | 状態 | 説明 |
|---|---|---|
| 白 | INCOMING（入電残像） | 初回入電・入電ゾーンに1.5秒間フェードアウト表示 |
| オレンジ | RETRY（入電残像） | 再入電・入電ゾーンに1.5秒間フェードアウト表示 |
| 黄 | QUEUE | 待機中（オペレータ空き待ち） |
| 緑 | SERVING | オペレータ対応中 |
| 灰 | DONE | 対応完了・完了ボックスに蓄積 |
| 赤 | ABANDONED | 放棄・放棄ボックスに蓄積 |

入電ゾーンのドット（INCOMING/RETRY）は、キュー入り直後に残像（`incomingTrail`配列）として
リアル時間1.5秒でフェードアウトする。シミュレーション速度に依存しない。

### ドットの移動フロー

```
[入電ゾーン残像] ──→ [QUEUE] ──→ [OPERATORS] ──→ [完了ボックス]
                       │                              （灰ドット蓄積）
                       │（満杯 or 待ちすぎ）
                       └──→ [放棄ボックス]
                                   │（赤ドット蓄積）
                                   └──→（30%が）再入電 → [入電ゾーン残像]にオレンジで再出現
```

ドットは目標座標（`tx`, `ty`）へ毎フレーム `update()` で滑らかに移動する（速度5px/frame）。

### 表示形式

上2行が新人（N）、下3行がベテラン（V）。

```

[V:●][V: ][V:●][V:●][V: ][V:●]
[V:●][V:●][V:●][V: ][V:●][V:●]
```

### 状態と色

| 表示 | 状態 | 色 |
|---|---|---|
| `[N: ]` | 新人・待機 | 灰 |

## メトリクス定義

| 指標 | 計算式 | 表示場所 |
|---|---|---|
| 入電数 | 総コール数（再入電含む） | フッター・サマリー |
| 受電数 | 対応完了したコール数 | フッター・サマリー |
| 放棄数 | 放棄されたコール数 | フッター・サマリー |
| 再入電数 | 再入電として発生したコール数 | フッター・サマリー |
| 応答率 | 受電数 ÷ 入電数 × 100 | フッター・サマリー |
| 稼働率（フッター） | 対応中人数 ÷ (総人数 - 休憩中人数) × 100（瞬間値） | フッター |
| 稼働率（サマリー） | 累積対応分 ÷ 累積稼働分 × 100（`servingMin / workMin`） | サマリー |
| CPH | 受電数 ÷ 経過時間（時） ÷ オペレータ数（1人あたり平均） | フッター |
| 平均待ち時間 | 待ち時間合計 ÷ (受電数 + 放棄数) | フッター・サマリー |
| 人件費 | 各オペレータの稼働時間（休憩除く） × 時給の累計 | フッター・サマリー |
| 1件あたりコスト | 総人件費 ÷ 受電数 | サマリー |
